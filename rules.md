# AI Project Rules

These rules define how the AI assistant should behave in this project.

---

## Scope

- Applicable to frontend projects
- Also applies when generating Git commit messages

---

## Global Rules (Always Apply)

- All Git commit messages MUST be written in **Simplified Chinese**
- Prefer clear, maintainable, and explicit code
- Do NOT over-engineer solutions
- Follow modular architecture principles

---

## Git Commit Rules

When generating commit messages:

- Use concise, descriptive wording
- Follow this format:

feat: 新增功能说明
fix: 修复问题说明
refactor: 重构说明

- ❗ Commit messages MUST be in Simplified Chinese

---

## Frontend / Mini Program Rules

When writing frontend or WeChat Mini Program code:

- Keep components small and focused
- Separate business logic from UI
- Avoid deeply nested structures
- Prefer readability over clever tricks

---

## When To Apply These Rules

Apply these rules when the user asks for:

- Generating git commit messages
- Writing frontend code
- Designing APIs
- Refactoring existing code
